# 📋 Техническое задание

> Шаг 2 цикла разработки

## Цель проекта

Разработать систему автоматизации обработки входящих заявок для отдела продаж компании Ilmavent:
- Telegram-бот для приёма и распределения заявок
- Веб-дашборд для вывода на экран в офисе
- Интеграция с Bitrix24

## Функциональные требования

### Модуль 1: Сбор заявок

Заявки поступают из 6 источников и должны попадать в единый Telegram-канал/бот:

| Источник | Способ получения |
|----------|------------------|
| Сайт ilmavent.ru | Вебхук с формы на сайте |
| Квиз 1 (ilmavent.ru/5questions) | Вебхук с квиза |
| Квиз 2 (ilmavent.ru/02) | Вебхук с квиза |
| Telegram (@ilmavent) | Telegram Bot API |
| WhatsApp (+7 925 473-53-36) | WhatsApp Business API |
| Почта (mail@ilmavent.ru) | IMAP-парсер входящих писем |

### Модуль 2: Telegram-бот

**Новая заявка приходит в Telegram:**
- Текст: «Новая заявка. Источник: [сайт/квиз/WhatsApp/почта/Telegram]»
- Минимум информации (без деталей типа площади дома)
- Кнопка: **«Взять заявку в работу»**

**При нажатии «Взять»:**
1. Проверка: может ли менеджер взять? (нет блокировок ↓)
2. Если да → в Bitrix24 **автоматически**:
   - Создаётся **карточка клиента** (контакт)
   - Менеджер назначается **ответственным**
   - Создаётся **задача «Установить контакт с клиентом»**
3. В Telegram-сообщении обновляется: «✅ Взял: Арсений | 14:32»
4. На дашборде появляется новый блок

**Проверка на дубли:**
- Перед созданием карточки: проверить номер телефона в Bitrix24
- Если номер найден → сообщение: «⚠️ Дубль. Заявка закреплена за [имя менеджера]»
- Новая карточка **не создаётся**

### Модуль 3: Ограничения менеджеров

| # | Условие | Действие |
|---|---------|----------|
| 1 | >7 красных задач в Bitrix24 | Кнопка «Взять» неактивна. Сообщение: «У вас больше 7 просроченных задач. Закройте задачи, чтобы брать новые заявки» |
| 2 | 3 раза попал в красную зону (6+ часов без контакта) | Блокировка на 1 неделю. Сообщение: «Вы заблокированы на 7 дней за просрочки» |
| 3 | Ручная блокировка от руководства | Админ нажимает кнопку → менеджер заблокирован на указанный срок |

**Порог «красной зоны» (6 часов) — настраиваемый параметр.** Заказчик планирует сточить позже.

### Модуль 4: Дашборд на ТВ

Веб-страница, открытая в браузере на полный экран (F11). Вид: блоки-карточки как у конкурентов на фото.

**Каждый блок-карточка содержит:**
- Название/тип заявки
- Источник (сайт/квиз/WhatsApp/почта)
- Время поступления
- Менеджер, взявший заявку
- Таймер (сколько прошло)

**Цветовая индикация блока:**

| Цвет | Время с момента взятия | Статус |
|------|----------------------|--------|
| 🟢 Зелёный (белый) | 0–2 часа | В работе, всё ОК |
| 🟡 Жёлтый | 2–6 часов | Внимание, скоро просрочка |
| 🔴 Красный | 6+ часов | Просрочено |

**Смена цвета происходит**, если менеджер:
- НЕ выполнил задачу «Контакт» в Bitrix24
- НЕ написал комментарий в сделке

Если задача выполнена и комментарий есть → блок **исчезает** с дашборда (или помечается как «обработано»).

**Автообновление:** каждые 30 секунд.

### Модуль 5: Уведомления

| Событие | Кому | Куда |
|---------|------|------|
| Новая заявка | Все менеджеры | Telegram-канал |
| Заявка перешла в жёлтый (2ч) | Менеджер, взявший | Личное сообщение в Telegram |
| Заявка перешла в красный (6ч) | Менеджер, взявший | Личное сообщение в Telegram |
| Менеджер заблокирован | Менеджер | Личное сообщение в Telegram |

### Модуль 6: Ночной режим

- С **00:00 до 06:00** заявки **не показываются** менеджерам в Telegram
- Заявки копятся и появляются в 06:00
- На дашборде могут отображаться как «ночные» (опционально)

### Модуль 7: Отчёты

Приходят в тот же Telegram-канал. Периодичность: **еженедельно** + **ежемесячно**. Структура одинаковая:

**Основные данные:**
- 📊 Всего лидов за период
- 👤 Кто сколько взял (Арсений — 3, Дмитрий — 7...)
- 🔴 Красные задачи: у кого сколько на момент отчёта
- ⚠️ Красная зона: кто сколько раз попал

**Дополнительно (если удастся вытянуть из Bitrix24):**
- Движение по воронке: сколько сделок на каждом этапе
- Переводы между этапами за период
- Сумма проданных сделок
- Распределение по менеджерам

## Нефункциональные требования

- Дашборд обновляется каждые 30 секунд
- Telegram-бот отвечает менее чем за 3 секунды
- Система работает 24/7
- Данные о блокировках и истории сохраняются в БД

## Входные данные

| Источник | Данные |
|----------|--------|
| Калькулятор (count_main) | Тип помещения, площадь, кухня, гостиная, потолки, этажи, санузлы, спальни, **телефон**, email |
| Попап «Скидка» | **Имя**, **телефон** |
| Форма «Получить смету» | **Имя**, **телефон**, email |
| Квиз /02 | Объект, площадь, системы, **имя**, **телефон** |
| Квиз /5questions | Помещение, площадь, проблема, приоритет, сроки, **имя**, **телефон** |
| WhatsApp / Telegram | Текст сообщения, номер телефона |
| Почта | Тема, текст письма, контакты отправителя |
| Bitrix24 | Статус задач, комментарии, воронка |

**Все формы собираются через Tilda и приходят на почту в формате Request details.**

### Этапы воронки Bitrix24

1. Переговоры
2. Проектирование
3. Проект закрыт (ЖДЕМ ДОМ)
4. Согласование сметы
5. Отложенная продажа
6. Сделка привлечение
7. Анализ причины проигрыша
8. Сделка успешна

| Сообщение в Telegram | Новая заявка с кнопкой «Взять» |

| Бэкенд | Node.js или Python |
