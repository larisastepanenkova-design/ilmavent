## Компоненты системы

### 1. Сбор заявок (leads/)
- Принимает вебхуки от сайта и квизов
- Парсит входящую почту по IMAP
- Получает сообщения из WhatsApp и Telegram
- Унифицирует формат заявки → передаёт в бота

### 2. Telegram-бот (bot/)
- Публикует заявку в канал с кнопкой «Взять»
- Обрабатывает нажатие кнопки
- Проверяет ограничения менеджера перед взятием
- Отправляет уведомления о просрочках
- Публикует отчёты

### 3. Bitrix24 интеграция (bitrix/)
- Создаёт карточку контакта
- Назначает ответственного менеджера
- Создаёт задачу «Установить контакт»
- Проверяет дубли по номеру телефона
- Читает данные воронки для отчётов

### 4. Система ограничений (managers/)
- Проверяет количество красных задач (>7 → блок)
- Считает попадания в красную зону (3 раза → блок на неделю)
- Поддерживает ручную блокировку от руководства
- Хранит историю в БД

### 5. Таймеры (timers/)
- Каждые 30 секунд: проверка цветовых зон (🟢→🟡→🔴)
- При смене цвета: отправка уведомления менеджеру
- Ночной режим: блокировка показа заявок 00:00–06:00
- Планировщик: еженедельный отчёт + ежемесячный отчёт

### 6. Дашборд (public/)
- Веб-страница с блоками-карточками
- Автообновление через API каждые 30 секунд
- Цветовая индикация по времени
- Адаптация под ТВ-экран (full screen)

### 7. База данных (db/)
- Таблица заявок (id, источник, время, менеджер, статус, цвет)
- Таблица менеджеров (id, telegram_id, bitrix_id, заблокирован, до_когда)
- Таблица истории (попадания в красную зону, блокировки)

## Схема базы данных

```
leads (заявки)
├── id
├── source (site / quiz1 / quiz2 / whatsapp / email / telegram)
├── client_name
├── client_phone
├── client_email
├── created_at
├── taken_at
├── taken_by (manager_id)
├── status (new / taken / contacted / expired)
├── color (green / yellow / red)
└── bitrix_deal_id

managers (менеджеры)
├── id
├── name
├── telegram_id
├── bitrix_user_id
├── is_blocked
├── blocked_until
├── blocked_reason
└── red_zone_count

history (история)
├── id
├── manager_id
├── event_type (red_zone / block / unblock / manual_block)
├── created_at
└── details
```
