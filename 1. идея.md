# 💡 Идея

> Шаг 1 цикла разработки

## Описание идеи

Система автоматизации обработки входящих заявок для отдела продаж **Ilmavent**. Три компонента:

1. **Telegram-бот** — заявки из всех источников (сайт, квизы, WhatsApp, почта, Telegram) попадают в общий канал. Менеджер нажимает «Взять» → автоматически создаётся карточка + задача в Bitrix24
2. **Дашборд на ТВ** — блоки-карточки заявок с цветовой индикацией (🟢→🟡→🔴), видно на экране в офисе в реальном времени
3. **Система контроля** — автоматические ограничители, уведомления о просрочках, еженедельные/ежемесячные отчёты

## Целевая аудитория

- **Руководитель отдела продаж (босс)** — видит в реальном времени, кто как работает. Больше не нужно вручную раздавать заявки и проверять обработку
- **Менеджеры (4 человека)** — получают заявки в Telegram, берут одной кнопкой, карточка в Bitrix создаётся автоматически
- **Старший менеджер** — освобождается от ручной раздачи заявок

## Какую проблему решает

**Сейчас:**
- Старший менеджер вручную раздаёт заявки → тратит время
- Менеджер сам создаёт карточку, сам ставит задачу → может забыть
- Нет контроля скорости обработки → заявки сливаются
- Босс не видит, кто работает, а кто нет → нет мотивации

**С системой:**
- Заявки приходят автоматически, менеджер берёт одной кнопкой
- Карточка + задача создаются автоматически в Bitrix24
- Босс видит на ТВ: кто взял, сколько времени прошло, кто просрочил
- Менеджер не может набрать заявок «в одну харю» и не обработать
- Лентяи видны сразу — красные блоки на экране перед всем офисом

## Основные возможности

### MVP (минимум)
- [ ] Telegram-бот с кнопкой «Взять заявку»
- [ ] Автосоздание карточки + задачи в Bitrix24
- [ ] Дашборд с цветовыми блоками на экран

### Полная версия
- [ ] Ночной режим (00:00–06:00 — заявки не показываются)
- [ ] Ограничитель: >7 красных задач → кнопка неактивна
- [ ] Ограничитель: 3 раза в красной зоне → блокировка на неделю
- [ ] Ручная блокировка менеджера
- [ ] Проверка дублей по номеру телефона
- [ ] Уведомления при просрочке (зелёная → жёлтая → красная)
- [ ] Еженедельные и ежемесячные отчёты в Telegram
- [ ] Статистика по воронке из Bitrix24

## Необходимые интеграции

| Сервис | Для чего |
|--------|----------|
| **Telegram Bot API** | Приём заявок, кнопка «Взять», уведомления, отчёты |
| **Bitrix24 REST API** | Создание карточек, задач, проверка дублей, чтение воронки |
| **Вебхуки с сайта/квизов** | Получение заявок с ilmavent.ru, квиз 1, квиз 2 |
| **WhatsApp Business API** | Получение заявок из WhatsApp |
| **Email (IMAP/SMTP)** | Получение заявок с почты mail@ilmavent.ru |

## Отличие от текущего процесса

| Было | Стало |
|------|-------|
| Заявка на почту → старший менеджер вручную раздаёт | Заявка → Telegram → менеджер сам берёт кнопкой |
| Менеджер сам создаёт карточку в Bitrix | Карточка создаётся автоматически |
| Нет контроля скорости | Цветовая система: зелёный → жёлтый → красный |
| Босс не видит кто работает | Дашборд на ТВ в реальном времени |
| Менеджер может сливать заявки | Автоматические ограничители + блокировки |
